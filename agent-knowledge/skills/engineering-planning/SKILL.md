## Overview

Guidelines for planning new features, creating implementation plans, and managing task progression.

## Core Capabilities

- **feature_breakdown**: Decomposing high-level requirements into technical tasks.
- **implementation_planning**: Documenting the "How" before writing code to ensure alignment.
- **risk_assessment**: Identifying potential bottlenecks or breaking changes early.

## Planning Workflow

1. **Requirements Gathering**: Understand the "Why" and "What" from the user.
2. **Technical Design**: Choose the right components, DTOs, and API endpoints.
3. **Artifact Creation**:
   - `implementation_plan.md`: The technical blueprint.
   - `task.md`: The progress checklist.
4. **Verification Plan**: Define how success will be measured (tests, manual verification).

## Best Practices

- Always review existing code for patterns before proposing a "new" way.
- Group tasks by component (API, Web, DB) for clarity.
- Update `walkthrough.md` periodically to show completed milestones.
